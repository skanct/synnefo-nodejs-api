<static> serverCreate(name, imageId, flavorId, optionals)

Create Server. This method allows you to create a new server. See the required parameters section below for an explanation of the variables that are needed to create a new server.
Parameters:
Name Type Description
name string Required, this is the name of the server
imageId string Required, this is the id of the image you would like to be used for the server
flavorId number Required, this is the id of the flavor you would like to be used for the server create
optionals Object Optional values
Properties
Name Type Description
personality Object[] A list of personality injections
personality[].path string The path (including name) for the file on the remote server. If the file does not exist, it will be created
personality[].contents string The data to be injected, must not exceed 10240 bytes and must be base64-encoded
personality[].owner string The file owner
personality[].group string The file group
personality[].mode number The file access mode
networks Object[] The networks attribute is not provided. the service will apply its default policy (e.g., automatic public network and IP assignment). If the network attribute is an empty list, the virtual server will not have any network connections
networks[].uuid string Provide an existing network ID. In that case, the virtual server will be connected to that network.
networks[].fixed_ip string Provide an existing network ID and an IP (which is already associated to that network). In that case, the virtual server will be connected to that network with this specific IP attached.
networks[].port string Provide an existing port ID to establish a connection through it.
metadata Object key:value pairs of custom server-specific metadata. There are no semantic limitations, although the OS and USERS values should rather be defined
project string The project where the VM is to be assigned. If not given, user’s system project is assumed (identified with the same uuid as the user)
